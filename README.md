# [AIBridge](https://aibridge-lilac.vercel.app/)

AIBridge is an end-to-end conversational AI infrastructure designed to seamlessly integrate intelligent, voice-capable chatbots into any business website. By leveraging advanced Retrieval-Augmented Generation (RAG) and multilingual voice synthesis, AIBridge enables organizations to instantly deploy highly contextual and responsive AI agents trained on their specific knowledge bases.

## About

The core objective of AIBridge is to bridge the gap between complex machine learning architectures and accessible business tooling. Instead of relying on static, scripted chatbots, AIBridge dynamically ingests a business's live website and internal documents to construct a deep semantic understanding of the company. It then provides an embeddable widget that can handle complex user inquiries via both text and real-time voice, supporting code-switched languages like Tinglish and Hinglish native to the Indian demographic.

## Architecture

The system is designed as a distributed, service-oriented architecture consisting of three primary tiers:

1. **AI/ML Microservice (Python / FastAPI)**: A high-performance inference and data processing pipeline responsible for document parsing, semantic chunking, embedding generation, vector search, and audio stream processing.
2. **Backend Gateway (Node.js / Express)**: The central orchestration layer handling authentication, business logic, relational data persistence, and session management. It acts as an asynchronous proxy between the frontend and the heavy ML workloads.
3. **Frontend Dashboard & Widget (Next.js)**: A comprehensive management portal for businesses to monitor and configure their AI, alongside the lightweight, vanilla JS embeddable widget injected into client sites.

## Features

- **Automated Knowledge Ingestion**: Crawls public URLs and parses uploaded documents (PDF, DOCX, TXT) to automatically build a domain-specific knowledge base.
- **Multimodal Interactions**: Supports both standard text chat and real-time voice conversations.
- **Multilingual Transliteration**: Natively understands and speaks mixed languages (Tinglish, Hinglish) using phonetic transliteration models.
- **Live Web Fallback**: Optionally searches the live internet to augment its knowledge when queries fall outside the provided corpus.
- **Embeddable Widget**: A lightweight, zero-dependency JavaScript snippet that injects the chat interface directly into any host website.

## Solution Engineering

The technical implementation of AIBridge involved solving several complex engineering challenges, particularly in the machine learning and audio processing domains.

### Machine Learning Infrastructure

The AI pipeline is the heaviest and most complex component of the system. 

**Retrieval-Augmented Generation (RAG) & Vector Database:**
When a business provides a URL or document, the Python microservice parses the raw text and applies semantic chunking. These chunks are passed through the `BAAI/bge-small-en-v1.5` embedding model to generate high-dimensional dense vectors. We utilized Qdrant as our vector database to store and index these embeddings. During inference, user queries are embedded and mapped against the Qdrant index using cosine similarity to retrieve the top-K most relevant context chunks.

**Live-Web Grounding:**
To prevent hallucinations on out-of-domain queries, we engineered a live-web fallback. If the Qdrant vector search yields low confidence, the pipeline asynchronously hits DuckDuckGo, scrapes the top public results, embeds them on the fly, and merges them into the LLM context window.

**Voice Pipeline (STT & TTS):**
The Voice Mode requires ultra-low latency. For Speech-to-Text (STT), we integrated Groq's hardware-accelerated `whisper-large-v3` model, which transcribes user audio streams in milliseconds. For Text-to-Speech (TTS), we integrated Microsoft Edge's Neural TTS engine. 

**Multilingual NLP Prompt Engineering:**
A significant challenge was supporting regional Indian languages in Voice Mode. The selected TTS model (`en-IN-NeerjaNeural`) operates phonetically on the English alphabet but fails to parse native Devanagari or Telugu scripts generated by standard LLMs. We engineered strict, multi-layered system prompts forcing the LLM (`llama-3.3-70b-versatile` / `llama-3.1-8b-instant`) to process the context and explicitly output its responses in transliterated Tinglish or Hinglish (A-Z characters only). This completely eliminated script-parsing failures and resulted in flawless, localized voice responses.

### Backend Orchestration

The Node.js backend serves as the traffic controller. Built with Express and TypeScript, it interfaces with a PostgreSQL database via the Prisma ORM. It handles:
- Authentication and session management.
- Relational mapping of Businesses, Widgets, and Chat Sessions.
- Persistence of all conversational history for analytics.

To ensure the Node.js event loop remains unblocked during heavy AI inference, the backend utilizes background workers to proxy requests to the Python microservice. The Next.js frontend then polls for the AI's response, preventing browser timeouts and maintaining a resilient connection even when the LLM takes several seconds to generate a complex RAG response.

### Frontend Application

The frontend is a Next.js 14 App Router application styled with TailwindCSS. It serves a dual purpose:
1. **The Management Dashboard**: A secure area where business owners can configure their widget, trigger new URL crawls, and manage their knowledge base.
2. **The Widget Provider**: It serves the `chatbot-widget.js` script. When injected into a client's site, this script dynamically constructs the chat interface (handling text, microphone streams, and audio playback) and proxies traffic securely back to our backend without exposing API keys.
